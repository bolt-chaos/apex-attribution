# Site data artifacts (`site/public/data/*.json`)

These JSON files are the **entire runtime backend** of the static site — no Python, no server. They are
generated by [`scripts/export_site.py`](../scripts/export_site.py) from the fitted models, which are
gitignored (~GBs), so **the JSON is committed** and regenerated locally when a model changes:

```bash
.venv/bin/python scripts/export_site.py     # writes this folder
```

CI (`.github/workflows/deploy.yml`) only builds the Vite app; it never runs Python.

## Two scales — do not mix

- **Main site** (car-swap, career arcs, H2H) lives on the **joint 2018–2025** model:
  `driver_skill = racecraft`, `car_pace = pace_r` (matches `build_scm_data.py --skill-source race`).
  Lower = faster for both.
- **Cross-era** ("Senna in a modern Red Bull") lives on the **1988–2025 sess_rw** model's single
  `skill` latent, with its own SCM fit and its **own mesh**. Its numbers are only comparable within
  `cross_era.json`.

`E[finish]` is marginalized over `circuit_type` (the site shows a circuit-average) and clamped to
[1, 20] in the UI.

## Files

### `names.json`
```ts
{ drivers: Record<string, string>; constructors: Record<string, string> }  // id -> display name
```

### `manifest.json`
```ts
{ generated: string; nDraws: number; meshN: number; expN: number;
  mainModel: string; crossEraModel: string; eras: string[];
  meshRanges: { skill: [number, number]; pace: [number, number] } }
```

### `drivers.json` — per-driver skill (joint scale)
```ts
Record<string /* driverId */, {
  name: string;
  seasons: number[];
  bySeason: Record<string /* year */, { lo: number; med: number; hi: number }>;  // career-arc band
  career: { lo: number; med: number; hi: number; draws: number[] };  // pooled; used for car-swap + H2H
}>
```
`draws` is `nDraws` (~200) downsampled posterior draws of the driver's career-pooled skill.

### `cars.json` — per team-year pace (joint scale)
```ts
Record<string /* "constructor@year" */, {
  constructor: string; year: number; name: string;
  lo: number; med: number; hi: number; draws: number[];   // pace_r draws
}>
```

### `finish_mesh.json` — E[finish] surface (joint scale)
```ts
{ skill_axis: number[];  // length meshN, ascending
  pace_axis: number[];   // length meshN, ascending
  z: number[][];         // z[i][j] = E[finish] at (skill_axis[i], pace_axis[j])
  note: string }
```
**Car-swap:** for a driver × car, take the paired `career.draws` × `cars.draws`, bilinearly
interpolate each pair on `z`, then report 5/50/95 quantiles as the credible band.

### `era.json` — car/driver split per era window (for the slider)
```ts
Array<{ label: string; start: number; end: number;
        carPct: number; driverPct: number;      // ICC variance shares (%), independent-roots graph
        carSpread: number; driverSpread: number  // interventional E[finish] spread (positions)
}>
```
The slider **snaps to these measured windows** (each an era-specific fit) — it does not interpolate
causal shares between them.

### `cross_era.json` — legends + dedicated mesh (1988-model scale)
```ts
{ mesh: { skill_axis: number[]; pace_axis: number[]; z: number[][] };   // its OWN scale
  modMean: number; modSd: number;                                       // modern field spread
  legends: Record<string /* legendId */, {
    name: string; peak: [number, number];
    eraMean: number; eraSd: number; rawDraws: number[];                 // raw skill draws (their era)
  }>;
  cars: Record<string /* "constructor@year" */, { name: string; pace: number }>;  // modern targets
  verstappen2024: number | null;  // reference skill for the plot line
  caveat: string }
```
**Cross-era query:** pick legend + target car. **Naive** = look up `rawDraws` directly on the
cross-era `mesh` at the car's `pace`. **Era-normalized** = map each draw
`equiv = modMean + (draw − eraMean) / eraSd · modSd`, then look up. Show both side by side with
`caveat` always visible.

### `teammates.json` — shared-teammate graph (full history, for the pathfinder)
```ts
{ nodes: Array<{ id: string; name: string }>;
  edges: Array<{ a: string; b: string; teams: string[] }> }  // a,b shared these team-years
```
BFS over `edges` gives the shortest shared-teammate path (e.g. Senna → … → Verstappen).

### `necessity.json` — rung-3 podium-necessity result (masthead hook + About explainer)
```ts
{ era: string; threshold: number; nPodiums: number;
  carPct: number; driverPct: number;   // % of podiums LOST but-for the car / the driver
  mostCarDependent: Array<{ name: string; pct: number }>;
  mostDriverDependent: Array<{ name: string; pct: number }> }
```
"Would this podium have happened *but for* the car / the driver?" — abduct luck, swap one factor
to mid-field, replay. The two percentages deliberately **do not sum to 100** (necessity isn't a
partition; most podiums needed both). Derived by `export_necessity()` in `export_site.py` from the
machine-readable `outputs/v2_attribution_2018_2025_joint.json` that `v2/attribution_v2.py` emits
alongside its text report (same joint 2018–2025 model as the main site) — re-run that script after
a refit and re-export; no hand-transcription.

### `incident_rates_2018_2025.json`, `reliability_rates.json`
Copied verbatim from `models/`. `incident_rates`: `Record<driverId, number>` + `_overall`.
`reliability`: `Record<constructorId, { p_mech_dnf: number; n_started: number }>` + `_overall`.
